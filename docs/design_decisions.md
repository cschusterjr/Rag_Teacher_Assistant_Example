# Design Decisions

## Why Markdown?

Markdown provides readable, structured curriculum documents that naturally support chunking.

## Why TF-IDF?

TF-IDF provides a simple, explainable retrieval baseline before introducing embeddings.

## Why Metadata?

Metadata allows curriculum-aware filtering before retrieval.

## Why Modular Architecture?

Separating ingestion, retrieval, generation, and evaluation makes the application easier to maintain and extend.

## Why Original Curriculum?

The curriculum corpus was written specifically for this project to avoid copyright concerns while simulating realistic instructional content.
